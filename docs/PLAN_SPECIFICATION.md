# プラン機能仕様書

## 概要

fanmark.idにおけるユーザープラン機能の包括的な仕様書です。プランの種類、制限、アップグレード・ダウングレード時の挙動について定義します。

> **関連仕様書**: [猶予期間管理仕様](./GRACE_PERIOD_SPECIFICATION.md) - ライセンス失効時の猶予期間と再取得制限について

## プラン種類と制限

### 基本プラン

| プラン | ファンマーク上限 | 期限延長 | 管理機能 | 想定ユーザー |
|--------|-----------------|----------|----------|-------------|
| **Free** | 3個 | 有料 | - | 個人利用 |
| **Creator** | 10個 | 有料 | - | コンテンツクリエイター |
| **Business** | 50個 | 有料 | - | 中小企業 |
| **Admin** | 無制限 | 無料 | 管理画面 | システム管理者 |

### 制限詳細

#### ファンマーク上限
- 新規取得時に上限チェックを実施
- 上限到達時は取得不可（エラーメッセージ表示）
- Admin は無制限（`limit = -1`）

#### 期限延長
- 各プランで有料課金による延長が可能
- Admin のみ無料で延長可能
- プラン対象外ファンマークは延長不可

## プラン変更仕様

### アップグレード時

**動作**:
- 即座にプラン変更完了
- 新プランの上限・機能が適用
- 既存ファンマークに影響なし

**例**: Free → Creator
- 上限が 3個 → 10個 に変更
- 追加取得が可能になる

### ダウングレード時

#### 1. 基本フロー

```
プラン変更要求
↓
現在所有数 vs 新プラン上限の比較
↓
所有数 ≤ 新プラン上限 → プラン変更完了
所有数 > 新プラン上限 → ファンマーク選択画面へ
```

#### 2. ファンマーク選択画面

**表示条件**: 現在所有数 > 新プラン上限

**画面仕様**:
- モーダル表示（キャンセル不可）
- 全ファンマークを一覧表示
- 新プラン上限数まで選択必須
- 選択状態の視覚的フィードバック
- 未選択ファンマークは「返却されます」として明示

**重要な注意事項**:
- この選択は一度限り（後から変更不可）
- 未選択ファンマークは即座に返却される（grace状態へ移行）
- 返却されたファンマークは猶予期間後に他ユーザーが取得可能になる

**例**: Business → Creator（20個所有 → 10個選択）
```
選択されたファンマーク（10個）:
- プラン対象として継続
- 期限延長可能

選択されなかったファンマーク（10個）:
- 即座に返却される（status: active → grace）
- 猶予期間（grace period）が設定される
- 猶予期間終了後、他ユーザーが取得可能になる
```

#### 3. 返却処理の詳細

ダウングレード時の返却処理は、個別の手動返却と同じフローを使用します:

1. **bulk-return-fanmarks Edge Function** が呼び出される
2. 各ファンマークに対して以下の処理が実行される:
   - ライセンスステータスを `active` → `grace` に変更
   - `license_end` を現在時刻に設定
   - `grace_expires_at` を計算・設定（現在時刻 + 猶予日数、次のUTC深夜0時に丸める）
   - `is_returned: true` を設定
   - 監査ログを記録
   - オーナーへの通知を送信
   - お気に入り登録者への通知を送信

> **参照**: 猶予期間の詳細は [猶予期間管理仕様](./GRACE_PERIOD_SPECIFICATION.md) を参照

## データベース設計

### fanmark_licenses テーブル

```sql
-- 主要カラム
id: uuid
fanmark_id: uuid
user_id: uuid
license_start: timestamp
license_end: timestamp
status: text ('active', 'grace', 'expired')
is_initial_license: boolean
is_returned: boolean                          -- 返却フラグ
grace_expires_at: timestamp                   -- 猶予期間終了日時
excluded_at: timestamp                        -- ライセンス失効日時
created_at: timestamp
updated_at: timestamp
```

### 状態管理

#### アクティブなファンマーク
```sql
status = 'active'
is_returned = false
grace_expires_at = null
excluded_at = null
```

#### 返却されたファンマーク（猶予期間中）
```sql
status = 'grace'
is_returned = true
grace_expires_at = '2025-01-16 00:00:00'  -- 次のUTC深夜0時
excluded_at = null
```

#### 失効済みファンマーク
```sql
status = 'expired'
is_returned = true
grace_expires_at = '2025-01-16 00:00:00'
excluded_at = '2025-01-16 00:00:00'
```

## 期限管理システム

### 基本フロー
```
active → grace (猶予期間) → expired
```

### ダウングレード時の返却処理
ダウングレード時に選択されなかったファンマークは、手動返却と同じフローで処理されます:

1. **即座にgraceステータスへ移行**
2. **猶予期間の設定**: `grace_expires_at = 現在時刻 + 猶予日数（次のUTC深夜0時に丸め）`
3. **通知の送信**: オーナーとお気に入り登録者に通知

### バッチ処理
- 既存の `check-expired-licenses` をそのまま利用
- ダウングレードで返却されたファンマークも通常の失効フローで処理
- 追加のバッチ処理は不要

## エラーハンドリング

### プラン変更時
- ネットワークエラー: 変更前の状態を保持
- 選択途中の離脱: プラン変更をロールバック
- データ整合性エラー: 管理者通知

### ファンマーク取得時
- 上限超過: 明確なエラーメッセージ
- Admin の場合: 上限チェックをスキップ
