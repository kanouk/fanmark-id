# プラン機能仕様書

## 概要

fanmark.idにおけるユーザープラン機能の包括的な仕様書です。プランの種類、制限、アップグレード・ダウングレード時の挙動について定義します。

> **関連仕様書**: [猶予期間管理仕様](./GRACE_PERIOD_SPECIFICATION.md) - ライセンス失効時の猶予期間と再取得制限について

## プラン種類と制限

### 基本プラン

| プラン | ファンマーク上限 | 期限延長 | 管理機能 | 想定ユーザー |
|--------|-----------------|----------|----------|-------------|
| **Free** | 3個 | 有料 | - | 個人利用 |
| **Creator** | 10個 | 有料 | - | コンテンツクリエイター |
| **Business** | 50個 | 有料 | - | 中小企業 |
| **Admin** | 無制限 | 無料 | 管理画面 | システム管理者 |

### 制限詳細

#### ファンマーク上限
- 新規取得時に上限チェックを実施
- 上限到達時は取得不可（エラーメッセージ表示）
- Admin は無制限（`limit = -1`）

#### 期限延長
- 各プランで有料課金による延長が可能
- Admin のみ無料で延長可能
- プラン対象外ファンマークは延長不可

## プラン変更仕様

### アップグレード時

**動作**:
- 即座にプラン変更完了
- 新プランの上限・機能が適用
- 既存ファンマークに影響なし

**例**: Free → Creator
- 上限が 3個 → 10個 に変更
- 追加取得が可能になる

### ダウングレード時

#### 1. 基本フロー

```
プラン変更要求
↓
現在所有数 vs 新プラン上限の比較
↓
所有数 ≤ 新プラン上限 → プラン変更完了
所有数 > 新プラン上限 → ファンマーク選択画面へ
```

#### 2. ファンマーク選択画面

**表示条件**: 現在所有数 > 新プラン上限

**画面仕様**:
- モーダル表示（キャンセル不可）
- 全ファンマークを一覧表示
- 新プラン上限数まで選択必須
- 選択状態の視覚的フィードバック
- 未選択ファンマークは「プラン対象外」として明示

**重要な注意事項**:
- この選択は一度限り（後から変更不可）
- 未選択ファンマークは期限延長不可になる
- 既存の期限までは利用可能

**例**: Business → Creator（20個所有 → 10個選択）
```
選択されたファンマーク（10個）:
- プラン対象として継続
- 期限延長可能

選択されなかったファンマーク（10個）:
- プラン対象外として設定
- 既存期限まで利用可能
- 期限延長不可
- 期限切れ時に自動失効
```

## データベース設計

### fanmark_licenses テーブル拡張

```sql
-- 既存カラム
id: uuid
fanmark_id: uuid
user_id: uuid
license_start: timestamp
license_end: timestamp
status: text ('active', 'grace', 'expired')
is_initial_license: boolean
created_at: timestamp
updated_at: timestamp

-- 新規追加カラム
plan_excluded: boolean DEFAULT false          -- プラン対象外フラグ
excluded_at: timestamp                        -- 対象外設定日時
excluded_from_plan: text                      -- ダウングレード元プラン
```

### 状態管理

#### プラン対象ファンマーク
```sql
plan_excluded = false
excluded_at = null
excluded_from_plan = null
```

#### プラン対象外ファンマーク
```sql
plan_excluded = true
excluded_at = '2025-01-15 10:30:00'
excluded_from_plan = 'business'
```

## UI/UX 仕様

### ダッシュボード表示

#### プラン対象ファンマーク
- 通常表示
- 期限延長ボタン表示
- 期限切れ警告（延長可能）

#### プラン対象外ファンマーク
```
[プラン対象外] 🔒 emoji_combination
期限: 2025-02-01 (延長不可)
```
- 「プラン対象外」バッジ表示
- 期限延長ボタン非表示
- 期限切れ警告（延長不可の注記付き）

### ファンマーク選択モーダル

#### レイアウト
```
┌─────────────────────────────────────┐
│ プラン変更に伴うファンマーク選択      │
├─────────────────────────────────────┤
│ 新プランでは10個まで保持可能です      │
│ ⚠️ この選択は後から変更できません    │
├─────────────────────────────────────┤
│ 選択: 7/10個                        │
│                                     │
│ ☑️ 🔥💯 [選択済み]                 │
│ ☑️ 🎮🚀 [選択済み]                 │
│ ⬜ 🌟📱 [プラン対象外になります]    │
│ ...                                 │
├─────────────────────────────────────┤
│           [確定] (3個不足)           │
└─────────────────────────────────────┘
```

#### インタラクション
- チェックボックスで選択/解除
- 選択数リアルタイム表示
- 確定ボタンは正確に上限数選択時のみ有効
- 未選択カードは視覚的に暗く表示

## 期限管理システム

### 基本フロー（既存）
```
active → grace (7日間) → expired
```

### プラン対象外の扱い
- **期限までの利用**: 通常通り可能
- **失効処理**: 既存フローをそのまま適用
- **延長制限**: UI で延長ボタン非表示のみ

### バッチ処理
- 既存の `check-expired-licenses` をそのまま利用
- プラン対象外も通常の失効フローで処理
- 追加のバッチ処理は不要

## エラーハンドリング

### プラン変更時
- ネットワークエラー: 変更前の状態を保持
- 選択途中の離脱: プラン変更をロールバック
- データ整合性エラー: 管理者通知

### ファンマーク取得時
- 上限超過: 明確なエラーメッセージ
- Admin の場合: 上限チェックをスキップ
